ипы вопросов из examples_cpc_gpc.md (что нужно покрыть)
Из файла видно 6 основных классов задач:

Позиції ВС / практика
Процедури / строки / допустимість (формальні вимоги)
Докази / стандарти (допустимість, експертиза, електронні докази)
Наслідки / резолютивна частина / виконання
ГПК: касація / відступ / виключна проблема
Уніфіковані запити: 10 рішень за 2 роки, pro/contra, “схожі факти”, чеклист дій
Список инструментов (MCP tools), которые нужны, чтобы отвечать на все вопросы
Ниже — минимальный “набор инструментов” (не один монолит), который покрывает все пункты.

1) Нормативка (коды/статьи/цитаты)
search_procedural_norms (у вас уже есть)
Зачем: Блок “B. Норма / правова рамка” для ЦПК/ГПК/КАС/КПК: сроки, требования, основания “без руху/повернення”, подсудность, полномочия суда и т.п.
Вход: procedure_code (cpc|epc|cac|crpc), query, опционально article, limit
Выход: список норм с реквизитами + короткими цитатами (без HTML)
2) Судебная практика ВС (ключевое для “позиції ВС”)
search_supreme_court_practice (нужен)
Зачем: покрывает все вопросы “позиция ВС/практика”, включая электронные доказательства, экспертизу, злоупотребление, обеспечительные меры, расходы, 3%/инфляция и т.д.
Вход (минимально):
procedure_code (cpc|epc|...)
query (теза)
time_range (например “last 2 years”)
court_level=SC (и/или палата: КЦС/КГС)
section_focus: COURT_REASONING, DECISION
limit (5–15)
Выход: список дел ВС с обязательными полями: суд/палата, дата, № справи, ссылки, короткие выдержки из мотивировочной части (без выдуманных реквизитов)
get_court_decision / get_case_text (нужен)
Зачем: когда нашли дело, нужно подтянуть полный текст/метаданные, чтобы извлечь корректные цитаты именно из COURT_REASONING/DECISION.
Вход: case_id или case_number + court + date (что есть)
Выход: структурированный текст по секциям + ссылка на источник
3) “Две линии практики” (pro/contra) и сравнение подходов
compare_practice_pro_contra (нужен)
Зачем: закрывает “практика «за» та «проти»”, “две линии”, “неоднозначність”.
Вход: query, procedure_code, time_range, опционально filters
Выход:
блок PRO: 3–7 дел с цитатами
блок CONTRA: 3–7 дел с цитатами
что различается (критерии, факты, процессуальные условия)
4) Поиск по “похожим фактам”
find_similar_fact_pattern_cases (нужен)
Зачем: вопрос “максимально схожі фактичні обставини” — это не просто теза, это similarity по фабуле.
Вход: facts_text, procedure_code, time_range, limit
Выход: топ похожих дел + коротко “почему похоже” (2–5 буллетов) + ссылки
Технически это обычно требует векторного индекса (Qdrant/embeddings) по корпусу решений.

5) Процедурные калькуляторы и чеклисты (строки, действия, риски)
calculate_procedural_deadlines (нужен)
Зачем: вопросы “який строк… і з якого моменту”, “строки оскарження ухвали/рішення”, “оголошення резолютивної vs отримання повного тексту”, “поновлення строку”.
Вход: procedure_code, event_type (рішення/ухвала/отримання повного тексту), event_date, appeal_type (апеляція/касація), опц. received_full_text_date, holidays_calendar
Выход: дата начала/конца, варианты расчета (если несколько точек отсчета), риски
build_procedural_checklist (нужен)
Зачем: “чеклист процесуальних кроків для подання позову/апеляції/касації”, “pre-flight перевірка”, “як уникнути повернення/без руху”.
Вход: procedure_code, stage (позов/апеляція/касація/забезпечення/витребування доказів/зустрічний позов), опц. case_category
Выход: чеклист + типовые основания отказа/возврата (норма + практика)
6) Денежные вопросы (ГПК: 3%/инфляция/неустойка/расходы)
calculate_monetary_claims (нужен)
Зачем: “3% річних та інфляційні”, “зменшення неустойки”, “судові витрати/гонорар”.
Вход: сумма, даты, тип требования, параметры расчета
Выход: расчет + правовая рамка + практика по ограничениям/мотивировке
7) “Пакетировщик ответа” (чтобы всегда соответствовать вашему шаблону)
format_answer_pack (нужен как финальный шаг)
Зачем: привести результаты из (1)-(6) к формату из answer_packaging_and_intents.md:
norm (норма + цитата)
position (практика ВС + цитаты)
conclusion (аналитический вывод)
risks (контраргументы/пастки)
Вход: результаты других tools + desired_output (thesis|checklist|comparison|collection|table)
Выход: готовый текст A–G по вашим правилам
Где именно “дырка” сейчас в вашем наборе
Есть: инструмент для нормативки (search_procedural_norms), т.е. закрыт блок B. Норма.
Не хватает: инструментов для практики ВС (поиск/получение текста/цитирование/две линии/схожие факты) и упаковщика под ваш шаблон.
